Introduction

This document explains how some PowerShell cmdlets—i.e. objects you can run from a terminal as commands—can be used to manage files in batch.

Command line use may not be popular among ordinary users, but once mastered, it does provides quite a few advantages the biggest one of which can be to avoid the tedium of manually managing large numbers of files.

A simple command such as Get-ChildItem, when used with judiciously selected parameters and piped through a formatting cmdlet, can quickly get the types of documents we want:

C:>\Users\john\Documents\Get-ChildItem -Recurse *.docx | Format-Table Fullname

Fullname
--------
C:\Users\john\Documents\Budget-2023.docx
C:\Users\john\Documents\Personal\Career\CV.docx
C:\Users\john\Documents\Personal\School\College\Alumni.docx
C:\Users\john\Documents\Recipes\Vegetarian\Mediterranean.docx
C:\Users\john\Documents\Reports\Annual-2019.docx
C:\Users\john\Documents\Reports\Quarterly-2021-1.docx
C:\Users\john\Documents\Reports\Quarterly-2020-4.docx
This can be achieved using File Explorer, as well. However, we have to navigate to the top folder, then enter the regular expression "*.docx" in a drop-down box, and—once the documents are located—stay in that pane.
